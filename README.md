# mindbridge
MindBridge is a youth mental health support platform frontend built with HTML5, CSS3, and vanilla JavaScript. The project provides a calm, accessible interface for young people to track their mood, complete self-assessment screenings, browse mental health resources, connect with peers in a moderated forum, and find professional support. The design emphasizes emotional safety through a soft blue-gray and navy color palette, generous whitespace, and warm micro-interactions such as heart-icon bookmarking. All pages share a persistent navigation bar, crisis hotline banner, and footer to ensure help is never more than one click away. The site is fully responsive, collapsing into a hamburger menu on mobile with ARIA attributes and scroll lock for accessibility. Interactive features include a Chart.js mood trend visualization, a step-by-step PHQ-9 and GAD-7 assessment flow with progress tracking, client-side resource filtering and search, and emoji-based mood logging. Since no backend is currently implemented, localStorage is used to simulate database functionality for user sessions, mood entries, assessment history, and bookmarks. This means data persists only on the user's current device and is cleared if browser storage is reset. The project is structured for future expansion into a MEN stack with MongoDB, Express, and Node.js using EJS templating to enable proper authentication, server-side validation, cross-device data sync, and secure storage of sensitive mental health information. To run the project locally, clone the repository and open index.html in a modern web browser, or serve the files using a local development server such as Live Server in VS Code. The Chart.js library is loaded via CDN for mood visualization, and all other dependencies are self-contained. File structure includes HTML pages in the root directory, CSS stylesheets in the styles folder, JavaScript files in the js folder, and image assets in the images folder. Planned improvements include backend integration with MongoDB for persistent storage, server-side form validation and password hashing, OAuth social login, dark mode toggle, prefers-reduced-motion support, image self-hosting with WebP compression, and a dynamic search bar across all content.
